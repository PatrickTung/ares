How it works Four steps to running a workflow
01 / FIND
Browse the catalogue
Search by domain, tool, or compute target. Each workflow has documentation, example inputs, and a citable DOI.
02 / CONFIGURE
Set your parameters
Clone the workflow repo and edit the params file. Every parameter is documented with defaults and valid ranges.
03 / RUN
Submit to compute
One command to run on Katana or Gadi. Workflows handles job scheduling, retries, and resource allocation automatically.
04 / CITE
Cite in your paper
Every workflow has a DOI via WorkflowHub. Copy the citation directly from the workflow page.
