Test workflow

Hello World

A minimal test workflow to confirm your ARES setup is working end-to-end. Requires no input data, completes in under 5 minutes, and writes a result file to your scratch directory. Run this first before trying any other ARES workflow.

Version
v1.0.0
Runtime
Bash · Docker
Walltime
< 5 min
Input data
None needed
Clicking the button below opens Katana OnDemand in your browser. You will be prompted to log in with your zID if you are not already. The job script will be pre-loaded — just click Submit to run.
▶ Run on Katana via OOD

What happens when you submit

  1. OOD opens the Job Composer with the ARES Hello World script pre-loaded.
  2. PBS schedules the job on a Katana compute node (usually starts within seconds for a 1-CPU job).
  3. The script prints system info and writes a result file to /srv/scratch/$USER/ares-hello-world/hello_world_result.txt
  4. Your run is logged to the ARES usage log.
  5. Job completes. Check the result file in your scratch directory or via OOD Files.

ARES does not currently have automated job submission for Gadi. Download the script below, edit your project code, and submit manually via ARE or SSH. You will need an active NCI account and a valid project allocation.
Open ARE (NCI) →

Steps

  1. Log in to are.nci.org.au with your NCI credentials.
  2. Open a Terminal session or use the Job Composer.
  3. Download and edit the script — replace YOUR_PROJECT with your NCI project code.
  4. Submit with qsub ares_hello_world_gadi.pbs

Gadi PBS script

#!/bin/bash
#PBS -N ARES_hello_world
#PBS -l ncpus=1
#PBS -l mem=1GB
#PBS -l walltime=00:05:00
#PBS -l storage=scratch/YOUR_PROJECT
#PBS -P YOUR_PROJECT
#PBS -j oe

OUTDIR="/scratch/$PROJECT/$USER/ares-hello-world"
mkdir -p "$OUTDIR"

cat > "$OUTDIR/hello_world_result.txt" << EOF
========================================
  ARES Hello World — Gadi — Success
========================================
Timestamp : $(date)
User      : $USER
Job ID    : $PBS_JOBID
Hostname  : $(hostname)
EOF

cat "$OUTDIR/hello_world_result.txt"
echo "Done. Result at: $OUTDIR/hello_world_result.txt"
Local runs use Docker. Install Docker Desktop (Windows or Mac) or Docker Engine (Linux) before proceeding. No other dependencies required.

Mac / Linux

# Download and run
curl -O https://raw.githubusercontent.com/patricktung-unsw/ares-pages/main/workflows/hello-world/local/run_local.sh
chmod +x run_local.sh
./run_local.sh

# Result will be at: ~/ares-hello-world/hello_world_result.txt

Windows (PowerShell)

# Download and run
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/patricktung-unsw/ares-pages/main/workflows/hello-world/local/run_local.ps1" -OutFile "run_local.ps1"
.\run_local.ps1

# Result will be at: ~\ares-hello-world\hello_world_result.txt
